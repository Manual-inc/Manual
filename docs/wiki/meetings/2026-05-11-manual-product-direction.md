# 2026-05-11 Manual 제품 방향 회의

출처: [2026-05-11 Manual 제품 방향 회의](../sources/2026-05-11-manual-product-direction.md)

## 회의 목적

Manual 데모를 기준으로 현재 구현 수준을 공유하고, 발표/후속 개발에 적합한 MVP 범위를 정한다.

## 핵심 결론

- Manual의 제품 핵심은 "실행 가능한 매뉴얼"이다.
- 현재는 CLI가 워크플로우를 만들고 macOS UI가 실행 상태와 로그를 모니터링하는 데모 단계다.
- 단기 MVP는 그래프 편집 CRUD 전체보다 [[partial-run-and-restart|부분 실행과 재시작]], [[node-storybook|노드 Storybook]], [[token-cost-observability|토큰 비용 관측]]에 집중한다.
- [[agent-sandboxing|에이전트 샌드박스]]는 중요하지만 우선 고급 옵션 또는 후속 범위로 둔다.
- [[agent-skill-routing|에이전트 스킬 지정]]은 Manual을 에이전트 전용 워크플로우 도구로 차별화하는 방향성이다.

## 논의 내용

### 현재 데모

- 노드는 크게 스크립트 노드와 에이전트 노드로 나뉜다.
- 에이전트 노드는 Claude, Codex, Pi, Hermes/Homecode 등 여러 실행자를 등록할 수 있다.
- 코드 리뷰 예시는 리뷰 컨텍스트 수집, clone, 테스트 실행, 병렬 리뷰, 더블 체크, 최종 요약/코멘트 흐름으로 구성된다.
- 현재 UI는 생성된 워크플로우와 로그를 보여주는 모니터링 성격이 강하다.
- CLI에는 노드 수정 기능이 있지만 UI에서는 아직 편집 기능이 중심이 아니다.

### 아키텍처

- 프로젝트 앱 서버가 JSON API로 워크플로우와 실행 상태를 관리한다.
- CLI와 네이티브 클라이언트가 동일 앱 서버와 통신한다.
- Codex CLI/Claude CLI 같은 로컬 CLI를 비대화형으로 호출하고 스트림 결과를 받아 처리한다.
- 현재는 MCP를 쓰지 않고 JSON API로 단순하게 간다.
- 자세한 정리는 [[manual-app-architecture|Manual 앱 아키텍처]]에 둔다.

### 자동화와 통제

- 전체 플로우 자동 실행도 필요하지만, 사용자가 단계별로 확인하고 중간에서 재시작하는 흐름이 더 중요하게 논의됐다.
- 앞 단계 전처리가 잘못됐는데 뒤 단계가 계속 도는 상황을 막아야 한다.
- 노드별 플레이 버튼, 실패 지점 재실행, 임의 입력 주입이 필요하다.

### 비용과 모델 배분

- 모든 단계를 고급 모델에 맡기는 대신 간단한 단계는 스크립트나 로컬 모델로 처리할 수 있다.
- 파일/메서드 의존성 분석 같은 전처리는 Gemma류 로컬 모델이나 정적 분석으로 처리하고, 요약된 정보를 고급 모델에 넘기는 방식이 언급됐다.
- 이 전략을 검증하려면 run/job 단위 토큰 및 비용 관측이 필요하다.

## 결정 사항

- [[2026-05-11-mvp-scope|2026-05-11 MVP 범위 결정]]을 생성해 단기 범위를 고정한다.
- 첫 개발 초점은 노드별 부분 실행/재시작이다.
- 두 번째 초점은 노드 Storybook/노드 단위 테스트 및 입력 주입이다.
- 세 번째 초점은 토큰/비용 사용량 기록과 비교다.
- 샌드박스는 중요하지만 당장 발표 핵심 범위에서는 옵션으로 다룬다.

## 액션 아이템

- 노드별 실행 버튼과 실패 지점 재실행 UX를 설계한다.
- 중간 입력을 임의로 넣고 노드를 독립 실행하는 테스트 화면을 설계한다.
- run/job별 토큰 사용량 저장 위치와 표시 방식을 정한다.
- 에이전트 노드 설정에 skill 지정 필드를 둘지 검토한다.
- 샌드박스 정책을 노드 옵션으로 표현하는 방식을 후속으로 검토한다.

## 열린 질문

- 그래프 편집 CRUD를 언제 UI 범위에 넣을 것인가?
- 노드 registry/schema를 CLI와 UI 중 어디에서 먼저 관리할 것인가?
- 토큰/비용 데이터는 어떤 provider/CLI별로 어떤 형식까지 표준화할 것인가?
- 스킬 강제 사용은 실제로 검증 가능한가, 아니면 프롬프트/설정 수준으로 둘 것인가?
