# 위키 목차

Manual 프로젝트 위키의 전체 목차다. 프로젝트 질문에 답할 때는 이 파일을 먼저 읽고 관련 페이지로 이동한다.

## 핵심 문서

- [개요](개요.md) — Manual 프로젝트의 현재 문제의식과 지식 지도를 요약한다.
- [작업 로그](작업-로그.md) — 위키 구조와 문서 변경 이력을 기록한다.

## 원천 요약

- [문제 정의](sources/문제-정의.md) — 프로젝트가 해결하려는 두 핵심 문제인 토큰 부족과 신뢰성 부족을 설명하는 원천 문서 요약.
- [2026-05-11 Manual 제품 방향 회의](sources/2026-05-11-manual-product-direction.md) — Manual MVP 범위, 앱 아키텍처, 부분 실행, 노드 Storybook, 토큰 비용 관측을 논의한 회의 요약.

## 문제

- [[토큰-부족]] — 고성능 LLM 사용 비용과 컨텍스트 한계가 만드는 토큰 부족 문제.
- [[신뢰성-부족]] — 확률적 모델과 불완전한 절차 준수가 만드는 신뢰성 부족 문제.
- [[실행-안전성-부족]] — 에이전트와 스크립트의 파일, 명령, 네트워크 접근을 안전하게 제한하기 어려운 문제.
- [[AI-도입-마찰]] — 토큰 비용과 리뷰 비용이 결합되어 AI 도입을 어렵게 만드는 문제.

## 개념

- [[토큰-배분]] — 제한된 토큰을 작업의 성격과 중요도에 따라 배분하는 개념.
- [[확률적-모델-리스크]] — LLM의 확률적 생성 방식에서 비롯되는 오류 가능성.
- [[단일-에이전트-오케스트레이션]] — 단일 에이전트 중심 작업 처리 방식과 그 한계.
- [[LLM-Harness]] — LLM을 다루기 위한 도구, 환경, 권한, 컨텍스트, 실행 제어를 포괄하는 기반 개념.

## 시스템

- [[매뉴얼-시스템]] — 토큰 효율성과 신뢰성을 함께 달성하기 위한 Manual의 전체 작업 시스템.
- [[매뉴얼-최적화-기능]] — MVP 중심 기능으로 AI 워크플로우의 토큰, 검증, 시간을 측정하고 개선 방향을 제안한다.
- [[매뉴얼-자기진화-기능]] — 단일 에이전트 매뉴얼이 최적화 결과와 검증 가드레일을 바탕으로 점진적으로 개선되는 기능.
- [[샌드박스-기능]] — 에이전트와 스크립트를 사전 정의된 파일, 명령, 네트워크 정책 안에서 실행하게 하는 안전 기능.
- [[문서-연결-코드]] — 새 코드가 관련 위키 문서와 연결되어 구현 이유와 도메인 맥락을 추적할 수 있게 하는 규칙.
- [[문서-규칙-테스트]] — 문서 작성 규칙을 실행 가능한 테스트로 검증하는 시스템.
- [[테스트-규칙]] — 문서 규칙 테스트, 기능 계약 테스트, 모듈 단위 테스트의 최상위 테스트 계층 규칙.
- [[기능-계약-테스트]] — `docs/usecase/*.feature` 행동 계약을 `manual-rs/crates/app-server/` 같은 각 앱과 엔드 구현에서 검증하는 테스트 규칙.

## 제품 기능

- [[Adaptive-Compute|Adaptive Compute]] — 워크플로우를 지속적으로 개선해 스크립트, 저비용 모델, 상위 LLM을 적절히 배치하고 비용을 최소화하는 원칙.
- [[partial-run-and-restart|부분 실행과 재시작]] — 노드별 실행, 실패 지점 재시작, 단계별 확인 기능.
- [[node-storybook|노드 Storybook]] — 노드 독립 실행, 임의 입력 주입, 노드 registry/schema 관리 기능.
- [[token-cost-observability|토큰 비용 관측]] — run/job 및 노드별 토큰/비용 기록과 최적화 기능.
- [[agent-skill-routing|에이전트 스킬 지정]] — 에이전트 노드에 skill을 지정하고 사용 여부를 관찰하는 기능.
