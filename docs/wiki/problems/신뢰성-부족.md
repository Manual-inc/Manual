# Reliability Deficit

Reliability deficit은 LLM 기반 에이전트의 결과를 시스템 차원에서 충분히 신뢰하기 어려운 문제다. 원천 문서는 [문제 정의](../sources/문제-정의.md)를 참고한다.

## Problem

LLM은 기본적으로 확률적으로 결과를 생성한다. 좋은 모델을 사용해도 오류 가능성은 남아 있으며, 절차 준수와 검증이 시스템으로 강제되지 않으면 결과 품질은 모델의 순간적인 판단에 크게 의존한다.

현재 Codex나 Claude Code 같은 도구는 대체로 [[단일-에이전트-오케스트레이션]]에 가깝다. 슬래시 커맨드, 스킬, 규칙 문서가 있어도 에이전트가 이를 생략하거나 불완전하게 따르면 확정적 보장이 되지 않는다.

## Failure Modes

- 정해진 절차가 있어도 항상 준수된다는 보장이 없다.
- 결과물이 모델의 순간적 판단에 크게 의존한다.
- 오류가 발생해도 어느 단계에서 왜 발생했는지 추적하기 어렵다.
- 사람이 최종 결과를 다시 꼼꼼히 리뷰해야 한다.
- 반복적인 수정과 검증 때문에 AI 도입 효과가 줄어든다.

## Related Concepts

[[확률적-모델-리스크]]는 이 문제의 모델 수준 원인이다. [[매뉴얼-시스템]]은 절차, 검증, 추적을 통해 이 문제를 완화하려는 시스템 수준 대응이다. [[LLM-Harness]]는 그 시스템이 LLM 실행을 다루는 기반 계층이다.

## Related Problems

Reliability deficit은 [[토큰-부족]]를 악화시킨다. 오류 수정과 재검증에 다시 토큰이 쓰이기 때문이다. 실행 환경 자체를 안전하게 제한하지 못하는 문제는 [[실행-안전성-부족]]으로 분리해 다룬다.
