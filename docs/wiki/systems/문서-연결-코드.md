# Documentation-Linked Code

Documentation-linked code is the convention that new code must carry a concise comment linking to the relevant `docs/wiki/` page that explains why the code exists.

## Purpose

Manual uses the `llm-wiki` structure to keep project documents consistent, cross-referenced, and periodically checked for accuracy. This convention extends that maintenance loop to code: code should not only describe what it does locally, but also point to the domain reason that justifies its existence.

## Rule

When adding new code at the repository root or in source directories, include a short comment with a link to the relevant wiki page.

The linked page should explain:

- why the code exists
- which problem, concept, or system behavior it supports
- how the code relates to neighboring wiki pages

## Expected Effect

Readers and future LLM agents should be able to follow the link from code into the wiki, then move through related pages such as [[매뉴얼-시스템]], [[LLM-Harness]], [[토큰-배분]], or [[신뢰성-부족]] to understand and verify the domain context.

## Relationship to Reliability

This convention supports [[매뉴얼-시스템]] by making implementation intent traceable. It also reduces [[신뢰성-부족]] because code review can compare implementation behavior against the documented domain reason instead of relying only on local code shape.
