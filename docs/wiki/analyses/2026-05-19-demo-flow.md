# 2026-05-19 Demo Flow

이 문서는 현재 저장소에서 Manual의 핵심 가치를 가장 빠르게 체감하는 데모 경로를 정리한다.

## 목적

고객이 "워크플로우를 실행한다"와 "실행 직후 optimization insight를 본다"를 한 번에 경험하게 하는 것이다.

현재 가장 짧은 경로는 `manual demo optimization`이며, 루트의 `scripts/demo-optimization.sh`는 그 CLI 진입점을 감싸는 얇은 wrapper다. 이 경로는 다음을 자동으로 수행한다.

1. `manual`과 `manual-app-server` 디버그 바이너리가 없으면 빌드한다.
2. 격리된 상태 디렉터리와 discovery 파일을 만든다.
3. 샘플 워크플로우를 등록한다.
4. `manual workflow run --human`으로 실행 로그와 최종 optimization report, optimization analysis를 함께 보여준다.

이 경로는 매 실행마다 새로운 demo workflow를 계속 쌓지 않고, `demo-optimization` workflow를 재사용한다. 따라서 반복 실행해도 workflow 목록이 불필요하게 오염되지 않는다.
또한 동일한 optimization report와 analysis를 두 번 반복해 출력하지 않고, `workflow run --human`의 완료 화면만으로 데모가 끝나게 유지한다.
또한 스크립트 wrapper는 기존 바이너리를 맹신하지 않고 현재 checkout 기준으로 `manual`과 `manual-app-server`를 다시 빌드해 최신 데모 동작을 보장한다.

## 사용 방법

```bash
manual demo optimization
```

또는 저장소 안에서는:

```bash
bash scripts/demo-optimization.sh
```

## 기대 효과

- workflow 실행과 optimization follow-through가 같은 데모 안에서 연결된다.
- CLI가 JSON 없이도 사람이 읽을 수 있는 형태로 현재 상태와 다음 행동을 보여준다.
- optimization 수치가 `reported`인지 `derived`인지 provenance를 같이 보여줘 신뢰도를 설명한다.
