# 문제 정의

이 문서는 Manual 프로젝트가 해결하려는 핵심 문제를 정리한 원천 문서 [docs/raw/문제-정의.md](../../raw/문제-정의.md)의 요약이다.

## Summary

Manual 프로젝트는 AI 에이전트를 실제 업무에 안정적으로 도입하기 어렵게 만드는 두 문제, [[토큰-부족]]와 [[신뢰성-부족]]를 해결하려고 한다.

토큰 부족은 OpenAI, Anthropic 등 LLM 제공업체의 고성능 모델 비용 상승과 긴 컨텍스트 사용 증가에서 비롯된다. 토큰을 많이 쓰는 방식은 비용과 한도 측면에서 지속 가능하지 않으며, 작업의 성격과 중요도에 따라 토큰을 적재적소에 사용하는 [[토큰-배분]]이 필요하다.

신뢰성 부족은 LLM이 확률적으로 결과를 생성한다는 사실과, 현재 Codex나 Claude Code 같은 도구가 대체로 [[단일-에이전트-오케스트레이션]] 방식으로 동작한다는 점에서 발생한다. 슬래시 커맨드, 스킬, 규칙 문서가 있어도 에이전트가 이를 생략하거나 불완전하게 따를 수 있다면 확정적인 보장이 되지 않는다. 이 문제를 줄이기 위해 [[매뉴얼-시스템]]이 필요하다.

## Key Contributions

- AI 업무 도입의 핵심 병목을 토큰 부족과 신뢰성 부족으로 정의한다.
- 토큰 비용과 리뷰 비용이 함께 증가하는 [[AI-도입-마찰]]을 설명한다.
- 모델 성능 향상만으로는 신뢰성 문제가 해결되지 않는다고 본다.
- 프로젝트 목표를 효율적인 토큰 사용과 신뢰할 수 있는 [[매뉴얼-시스템]] 구축으로 둔다.

## Terminology Note

원천 문서에는 "하네스 시스템"이라는 표현이 등장하지만, 이후 위키에서는 용어를 분리했다. [[LLM-Harness]]는 LLM을 다루기 위한 도구, 환경, 권한, 컨텍스트, 실행 제어를 포괄하는 기반 개념이고, [[매뉴얼-시스템]]은 토큰 효율성과 신뢰성을 달성하기 위한 더 넓은 작업 시스템이다.
