# 부분 실행과 재시작

출처: [2026-05-11 Manual 제품 방향 회의](../sources/2026-05-11-manual-product-direction.md)

## 개념

워크플로우 전체를 한 번에 실행하는 것뿐 아니라, 특정 노드 또는 실패 지점부터 다시 실행할 수 있게 하는 기능이다. Manual의 자동화가 "켜놓고 기다리는 블랙박스"가 아니라 사용자가 단계별로 관찰하고 개입할 수 있는 실행 도구가 되게 한다.

## 필요한 동작

- 노드별 플레이 버튼을 제공한다.
- 특정 노드부터 실행한다.
- 실패한 노드부터 재시작한다.
- 실행 중인 흐름을 멈출 수 있다.
- 이전 결과를 확인한 뒤 다음 단계로 넘긴다.
- 중간 노드 실행에 필요한 입력값을 직접 넣을 수 있다.
- 전체 자동 실행과 단계별 수동 진행을 모두 지원한다.

## 해결하려는 문제

- 앞 단계 전처리가 잘못됐는데 뒤 단계가 계속 실행되는 문제.
- 에이전트가 잘못된 방향으로 오래 실행되며 토큰과 시간을 낭비하는 문제.
- 실패 후 전체 플로우를 처음부터 다시 돌려야 하는 문제.
- 실행 결과를 신뢰하기 전에 단계별 검증이 어려운 문제.

## UX 메모

- 노드에 hover하면 실행 버튼을 노출하는 안이 언급됐다.
- 실패 상태는 빨간색 등 명확한 시각 상태로 표시한다.
- 재실행 시 기존 입력/출력과 새 입력을 비교할 수 있으면 좋다.
