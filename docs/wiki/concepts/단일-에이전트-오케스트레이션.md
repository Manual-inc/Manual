# Single-Agent Orchestration

Single-agent orchestration은 하나의 에이전트가 작업 이해, 계획, 실행, 검증을 대부분 직접 조율하는 방식이다.

## Definition

현재 Codex나 Claude Code 같은 도구는 사용자가 지시를 주면 단일 에이전트가 전체 작업을 오케스트레이션하는 형태에 가깝다. 슬래시 커맨드, 스킬, 규칙 문서가 보조 장치로 존재할 수 있지만, 에이전트가 항상 그것들을 완전하게 따르는 것은 아니다.

## Limitation

단일 에이전트가 절차 준수 여부까지 스스로 판단하면, 절차가 생략되거나 검증이 약해질 수 있다. 이는 [[신뢰성-부족]]의 원인이 된다.

## Related System Direction

[[매뉴얼-시스템]]은 단일 에이전트의 판단에만 의존하지 않고, 실행 절차와 검증 조건을 외부 시스템으로 보강하려는 방향이다. [[LLM-Harness]]는 이때 도구, 환경, 권한, 컨텍스트 같은 실행 기반을 제공하는 별도 개념이다.
