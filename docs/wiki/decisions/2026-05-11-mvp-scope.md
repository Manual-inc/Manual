# 2026-05-11 MVP 범위 결정

출처: [2026-05-11 Manual 제품 방향 회의](../sources/2026-05-11-manual-product-direction.md)

## 결정

단기 MVP는 그래프 편집 CRUD 전체가 아니라 다음 세 축에 집중한다.

1. [[partial-run-and-restart|부분 실행과 재시작]]
2. [[node-storybook|노드 Storybook]]
3. [[token-cost-observability|토큰 비용 관측]]

[[agent-sandboxing|에이전트 샌드박스]]는 중요하지만 고급 옵션 또는 후속 개발 범위로 둔다. [[agent-skill-routing|에이전트 스킬 지정]]은 제품 방향성에 포함하되, MVP 핵심 범위와 분리해서 검토한다.

## 배경

현재 데모는 CLI가 워크플로우를 생성하고 UI가 실행 상태를 모니터링하는 수준이다. 여기서 UI CRUD를 모두 넣으면 범위가 커질 수 있다. 반면 부분 실행, 노드 독립 테스트, 비용 관측은 에이전트 워크플로우 도구로서의 차별점을 직접 보여준다.

## 근거

- 부분 실행은 사용자가 자동화 흐름을 신뢰하고 통제할 수 있게 한다.
- 노드 Storybook은 노드가 독립 실행 가능한 단위라는 제품 모델을 분명하게 보여준다.
- 토큰/비용 관측은 "비싼 모델에 모든 일을 맡기지 않는다"는 전략을 검증할 수 있게 한다.
- 샌드박스는 강력하지만 설명과 구현 범위가 크므로 MVP 첫 화면의 핵심 메시지에서는 뒤로 뺄 수 있다.

## 결과

- 그래프 편집 기능은 필요하지만 지금 당장 핵심 발표 범위는 아니다.
- 실행 관찰에서 실행 통제로 제품 경험을 한 단계 확장한다.
- 이후 노드 registry, skill 지정, sandbox 정책이 자연스럽게 이어질 수 있다.
