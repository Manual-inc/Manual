# Manual Overview

Manual은 AI 에이전트를 실제 업무에 도입할 때 발생하는 [[토큰-부족]], [[신뢰성-부족]], [[실행-안전성-부족]]을 함께 해결하려는 [[매뉴얼-시스템]] 프로젝트다.

핵심 문제의식은 단순하다. 고성능 LLM은 강력하지만 비싸고, 많은 토큰을 요구하며, 결과가 항상 신뢰 가능한 방식으로 생성되지는 않는다. 이 때문에 사용자는 AI에게 일을 맡기면서도 결과를 다시 검토하고 수정해야 하며, 이 과정에서 토큰 비용과 사람의 시간이 동시에 소모된다. 자세한 원천 주장은 [문제 정의](sources/문제-정의.md)에 정리되어 있다.

## Current Thesis

Manual의 초기 가설은 다음과 같다.

1. 토큰은 무한한 자원이 아니라 제한된 실행 자원으로 다뤄야 한다.
2. LLM의 확률적 오류 가능성은 모델 선택만으로 제거되지 않는다.
3. 단일 에이전트 중심 오케스트레이션은 절차 준수와 검증을 확정적으로 보장하지 못한다.
4. 에이전트와 스크립트 실행은 사전에 정의된 파일, 명령, 네트워크 경계 안에서 이루어져야 한다.
5. 따라서 필요한 곳에 토큰을 배분하는 [[토큰-배분]], 결과를 검증 가능한 단계로 제약하는 [[매뉴얼-시스템]], 실행 경계를 제한하는 [[샌드박스-기능]]이 함께 필요하다.

## MVP Direction

현재 MVP 중심 문서는 [[매뉴얼-최적화-기능]]이다. 이 기능은 AI 워크플로우의 토큰 사용량, 검증 상태, 소요 시간을 측정하고, 측정 결과를 바탕으로 개선 방향을 제안한다.

2026-05-11 제품 방향 회의에서는 단기 제품 범위가 [[partial-run-and-restart|부분 실행과 재시작]], [[node-storybook|노드 Storybook]], [[token-cost-observability|토큰 비용 관측]]으로 구체화됐다. 이 결정은 [[2026-05-11-mvp-scope|2026-05-11 MVP 범위 결정]]에 정리되어 있다.

제품 아키텍처는 [[manual-app-architecture|Manual 앱 아키텍처]]에 정리된 것처럼 앱 서버를 중심에 두고 CLI와 네이티브 클라이언트가 동일한 워크플로우와 실행 상태를 공유하는 로컬-first 구조로 출발한다. 이 구조 위에서 Manual은 "실행 가능한 매뉴얼"이라는 제품 모델을 구현한다.

확장 방향으로는 [[매뉴얼-자기진화-기능]]이 있다. 이 기능은 단일 에이전트 매뉴얼로 시작한 작업 시스템이 최적화 결과와 검증 가드레일을 바탕으로 점진적으로 더 효율적인 매뉴얼로 진화하게 만든다.

제품 원칙으로는 [[Adaptive-Compute|Adaptive Compute]]가 있다. 이는 결정적으로 처리 가능한 과제는 스크립트로, 낮은 위험의 비결정적 과제는 로컬 LLM 또는 저비용 모델로, 핵심적이고 고심이 필요한 비결정적 과제는 상위 LLM으로 라우팅하면서 워크플로우 비용을 지속적으로 낮추는 방향이다.
