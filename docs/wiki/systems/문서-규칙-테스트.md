# 문서 규칙 테스트

문서 규칙 테스트는 `docs/` 하위 문서가 `llm-wiki` 구조를 계속 지키는지 확인하는 실행 가능한 검증 코드다.

## Purpose

Manual은 문서가 서로 연결되어 있어야 코드와 도메인 지식을 함께 검증할 수 있다. 고아 문서가 생기면 [[문서-연결-코드]] 규칙과 [[매뉴얼-시스템]]의 추적성이 약해진다.

## First Check

첫 번째 테스트는 `docs/raw`와 `docs/wiki`의 모든 Markdown 문서에 들어오는 링크가 하나 이상 있는지 검사한다. 들어오는 링크가 없는 문서가 있으면 테스트는 실패하고 해당 파일명을 출력한다.

## Scope

- 대상: `docs/raw/**/*.md`, `docs/wiki/**/*.md`
- 추가 대상: `docs/usecase/**/*.feature`의 위키 링크 주석
- 제외: 없음
- 링크 형식: Obsidian wiki link와 Markdown `.md` 링크
- 유스케이스 추적 링크 형식: `# wiki: docs/wiki/...`

## Usecase Link Check

`docs/usecase/*.feature` 파일은 관련 위키 문서와 주석으로 연결되어야 한다. 이 연결은 문서 내용이 바뀔 때 어떤 행동 계약을 함께 확인해야 하는지 찾기 위한 추적성 장치다.

규칙은 다음과 같다.

- 각 `.feature` 파일은 상단 근처에 하나 이상의 `# wiki: docs/wiki/...` 주석을 가져야 한다.
- `# wiki:` 주석은 해당 유스케이스의 근거가 되는 `docs/wiki/` 문서를 가리켜야 한다.
- 문서 규칙 테스트는 주석이 가리키는 위키 문서가 실제로 존재하는지 검사한다.
- 관련 위키 문서가 변경되면 연결된 `.feature` 파일을 검토해 시나리오가 여전히 일관적인지 확인해야 한다.

## Relationship

이 테스트는 [[위키-스키마]]의 lint checklist 중 `docs/` 문서 구조와 추적 링크 검사를 실행 가능한 코드로 옮긴다. 전체 테스트 계층 안에서의 위치는 [[테스트-규칙]]을 참고한다. `docs/usecase/*.feature`의 행동 계약 구현 테스트는 [[Feature-Contract-Tests]]에서 별도로 다룬다.
