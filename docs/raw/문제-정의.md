# 문제 정의

이 프로젝트는 AI 에이전트를 실제 업무에 안정적으로 도입하기 어렵게 만드는 두 가지 핵심 문제를 해결하기 위해 시작한다.

1. 토큰 부족
2. 신뢰성 부족

이 두 문제는 서로 분리된 불편이 아니라, 함께 작동하면서 AI 도입의 실질적인 비용과 리스크를 키운다. 토큰은 비싸고 제한적이며, 에이전트의 결과는 충분히 검증되지 않으면 반복적인 오류 수정과 리뷰 비용을 만든다. 결국 AI를 사용했음에도 업무 속도와 품질이 기대만큼 개선되지 않고, 토큰 비용과 사람의 검토 시간이 동시에 소모되는 상황이 발생한다.

## 1. 토큰 부족

최근 OpenAI, Anthropic과 같은 LLM 제공업체의 고성능 모델은 점점 더 높은 비용으로 제공되고 있다. 모델의 성능이 좋아질수록 더 많은 작업을 맡길 수 있지만, 그만큼 토큰 사용량과 비용도 빠르게 증가한다.

이 상황에서 단순히 더 많은 컨텍스트를 넣고, 더 큰 모델을 사용하고, 더 긴 대화를 유지하는 방식은 지속 가능하지 않다. 특히 실제 업무에서는 다음과 같은 문제가 반복된다.

- 작업마다 과도한 컨텍스트가 함께 전달된다.
- 중요하지 않은 정보에도 고가 모델의 토큰이 사용된다.
- 같은 내용을 여러 에이전트 또는 여러 단계에서 반복해서 소비한다.
- 토큰 한도 때문에 필요한 정보를 충분히 유지하지 못한다.
- 비용 부담 때문에 AI 사용 범위가 제한된다.

따라서 토큰은 단순히 많이 쓰는 자원이 아니라, 작업의 성격과 중요도에 따라 적재적소에 배분되어야 하는 제한 자원이다.

## 2. 신뢰성 부족

아무리 좋은 모델을 사용하더라도 LLM은 기본적으로 확률적으로 답을 생성한다. 따라서 모델 자체만으로는 항상 올바른 결과를 보장할 수 없다. 적절한 시스템, 검증 절차, 실행 하네스가 마련되어 있지 않으면 오류가 발생할 가능성은 계속 남는다.

현재 Codex나 Claude Code 같은 도구의 사용 방식은 대체로 단일 에이전트가 전체 작업을 오케스트레이션하는 형태에 가깝다. 슬래시 커맨드, 스킬, 규칙 문서 같은 보조 장치가 있더라도, 에이전트가 상황에 따라 이를 생략하거나 불완전하게 따를 수 있다면 시스템 차원의 확정적인 보장은 되지 않는다.

이로 인해 다음과 같은 문제가 발생한다.

- 정해진 절차가 있어도 에이전트가 항상 준수한다는 보장이 없다.
- 결과물의 품질이 모델의 순간적인 판단에 크게 의존한다.
- 오류가 발생해도 어느 단계에서 왜 발생했는지 추적하기 어렵다.
- 사람이 최종 결과를 다시 꼼꼼히 리뷰해야 한다.
- 반복적인 수정과 검증 때문에 AI 도입 효과가 줄어든다.

신뢰성 부족은 단순히 "모델이 가끔 틀린다"는 문제가 아니다. 실제 업무에서는 오류를 발견하고, 원인을 찾고, 수정하고, 다시 검증하는 비용이 누적된다. 이 비용이 커지면 AI를 도입했음에도 전체 생산성은 오히려 낮아질 수 있다.

## 결과적으로 발생하는 문제

토큰 부족과 신뢰성 부족이 결합하면 다음과 같은 악순환이 만들어진다.

1. 더 좋은 결과를 얻기 위해 더 많은 토큰과 더 비싼 모델을 사용한다.
2. 그러나 신뢰할 수 있는 실행 및 검증 체계가 부족해 오류가 발생한다.
3. 오류를 잡기 위해 사람이 더 많은 시간을 리뷰와 수정에 쓴다.
4. 추가 수정 과정에서 다시 토큰과 비용이 사용된다.
5. 결과적으로 비용은 증가하지만, 업무 품질과 속도는 충분히 개선되지 않는다.

이 악순환은 AI를 실제 업무에 도입하는 데 가장 큰 장벽이 된다. AI가 유용하다는 가능성은 분명하지만, 비용 효율성과 신뢰성을 동시에 확보하지 못하면 조직과 개인은 AI를 핵심 업무에 안심하고 맡기기 어렵다.

## 프로젝트의 목표

이 프로젝트의 목표는 제한된 토큰을 효율적으로 사용하면서도, 결과의 신뢰성을 높일 수 있는 시스템을 구축하는 것이다.

이를 위해 프로젝트는 다음 방향을 지향한다.

- 토큰을 작업의 성격과 중요도에 따라 적재적소에 사용한다.
- 불필요한 컨텍스트 반복과 고비용 모델 사용을 줄인다.
- 에이전트가 정해진 절차를 안정적으로 따르도록 하네스 시스템을 구축한다.
- 결과물을 검증 가능한 단계와 산출물로 나눈다.
- 모델의 확률적 오류 가능성을 시스템 차원의 검증과 제약으로 보완한다.
- 사람이 모든 결과를 다시 검토해야 하는 부담을 줄인다.

궁극적으로 이 프로젝트는 "더 많은 토큰을 쓰는 AI"가 아니라, "필요한 곳에 토큰을 쓰고, 검증 가능한 방식으로 신뢰할 수 있는 결과를 만드는 AI 작업 시스템"을 목표로 한다.
