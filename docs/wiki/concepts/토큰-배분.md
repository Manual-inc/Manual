# Token Allocation

Token allocation은 제한된 토큰을 작업의 성격과 중요도에 따라 적재적소에 배분하는 개념이다.

## Definition

토큰은 단순히 많이 쓸수록 좋은 자원이 아니라 제한된 실행 자원이다. Manual은 모든 작업에 동일한 수준의 컨텍스트와 모델 비용을 쓰는 대신, 단계별 중요도와 실패 비용에 따라 토큰 사용량을 조절해야 한다는 관점을 가진다.

## Design Implications

- 작업 단계별 토큰 예산이 필요하다.
- 고비용 모델은 판단, 설계, 검증처럼 높은 품질이 필요한 단계에 우선 배정되어야 한다.
- 반복적으로 쓰이는 정보는 매번 다시 소비하지 않고 구조화된 wiki나 요약 산출물로 재사용해야 한다.
- 불필요한 컨텍스트 전달을 줄이는 검색, 요약, 참조 전략이 필요하다.
