# Token Scarcity

Token scarcity는 고성능 LLM을 실제 업무에 사용할 때 토큰 비용과 컨텍스트 한도가 핵심 제약이 되는 문제다. 원천 문서는 [문제 정의](../sources/문제-정의.md)를 참고한다.

## Problem

OpenAI, Anthropic 같은 LLM 제공업체의 고성능 모델은 강력하지만 비싸다. 모델이 좋아질수록 더 많은 일을 맡길 수 있지만, 실제 업무에서는 긴 컨텍스트, 반복된 지시, 중복된 파일 읽기, 여러 단계의 재시도 때문에 토큰 사용량이 빠르게 증가한다.

## Failure Modes

- 작업마다 과도한 컨텍스트가 함께 전달된다.
- 중요하지 않은 단계에도 고가 모델의 토큰이 쓰인다.
- 같은 정보가 여러 에이전트나 여러 단계에서 반복 소비된다.
- 토큰 한도 때문에 필요한 정보를 유지하지 못한다.
- 비용 부담 때문에 AI 사용 범위가 제한된다.

## Related Concepts

[[토큰-배분]]은 이 문제에 대한 핵심 대응 개념이다. 토큰을 많이 쓰는 것이 아니라, 필요한 곳에 배분하는 방식이 필요하다.
